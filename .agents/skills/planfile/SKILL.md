# Why Planfile
**If we cannot agree to or manage the plan, we cannot create solutions.**

# Usage

Use this file to keep track of tasks, completed, in progress and todo.

Tasks are **individual** work items. Do not wrap complex work in a single task. Break it down.

Use Markdown checklist syntax:
```
- [ ] This task is TODO
- [x] This task is completed
```

And be aware of the following modifiers at the end of each line:
```
- [ ] This TODO is in progress 🚧
- [ ] This TODO has high priority 🔺
```

Mark tasks as in progress (🚧) **only** when you begin working them.

When you complete a task, add the completion date in YYYY-MM-DD format:
```
- [x] This task is completed ✅ 2026-04-17
```

# Priority

In general, the plan is in order of what to accomplish next, but make sure to check the modifiers. If you are unsure, ask.

Do not return to previously completed tasks unless instructed.

Do not stray from an in progress task until it is completed unless instructed.

# Safety

Ask before adding or removing a task from the plan.

Ask for confirmation before marking a task as complete.
